# 深模块

出自《软件设计的哲学》：

**深模块** = 小接口 + 大量实现

```
┌─────────────────────┐
│     小接口          │  ← 方法少，参数简单
├─────────────────────┤
│                     │
│                     │
│    深层实现         │  ← 复杂逻辑被隐藏
│                     │
│                     │
└─────────────────────┘
```

**浅模块** = 大接口 + 少量实现（应避免）

```
┌─────────────────────────────────┐
│         大接口                  │  ← 方法多，参数复杂
├─────────────────────────────────┤
│    薄层实现                     │  ← 只是透传
└─────────────────────────────────┘
```

设计接口时，问自己：

- 能否减少方法数量？
- 能否简化参数？
- 能否把更多复杂性隐藏在内部？
